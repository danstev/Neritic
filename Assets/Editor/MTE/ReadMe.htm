User Manual

Mesh Terrain Editor(MTE) has two main functionality:

  • MTE Converter that can convert Unity3D’s terrain object into a MTE-editable mesh-terrain.
  • MTE Editor that can edit a MTE-editable mesh-terrain, including height tools, texture tool and two simple mesh tools.

MTE-editable mesh-terrain will be called mesh-terrain for short below.

Creating Mesh-terrains

You can convert your mesh or Unity’s terrain object into a mesh-terrain, which is just a mesh with special terrain shader. MTE provides some single-pass shaders with 2/3/4/5 splat textures and two simple vertex-colored shaders.

Convert Unity’s terrain object into a mesh-terrain

If you got a Unity terrain object, you can convert it into a mesh-terrain using the MTE converter follows these steps:

  1. At the menu Window/MTE/Converter, open the MTE Converter and Choose Terrain.
  2. Choose your terrain GameObject(the one with a Terrain component). The number of splat textures of the terrain object should be 2/3/4/5. You can only use 4 splat textures in free version.
  3. Set name of the converted mesh.
  4. Set output directory.
  5. Press Begin Convert at the bottom and wait a moment.
  6. When it is done, the mesh-terrain is placed in the output dir. And the instanced prefab of converted mesh-terrain is in the hierarchy.

Then you can use the MTE Editor to edit the generated mesh-terrain.

Note MTE have already adjusted the tilling of splat textures to fit the original terrain for you. But if it doesn’t meet your requirement, you can adjust that whenever you like.

Convert a existing mesh into a mesh-terrain

If you got a terrain mesh that is made with 3DSMax or other modeling software, you can convert it into a mesh-terrain using the MTE converter follows these steps:

  1. Import your terrain mesh file into a project. Drag it into the hierarchy.
  2. At the menu Window/MTE/Converter, open the MTE Converter and Choose Mesh.
  3. Choose your terrain mesh GameObject(the one with a MeshRenderer)
  4. Set name of the converted mesh.
  5. Set output directory(Choose one inside the Assets directory).
  6. Choose Create prefab if you want to save the mesh-terrain as a prefab, otherwise your mesh GameObject will be converted into a mesh-terrain locally in the scene. It’s recommended to create a prefab for the mesh-terrain.
  7. Set the expected splat texture count. You can only use 2/3/4/5 splat textures. You can only use 4 splat textures in free version.

    Note: The number of splat texture can be modified after the conversion. You can change the shader to achieve that but there maybe some extra operations. Please try it yourself.

    Note: The actual number of control textures is determined by the splat texture count. 2/3/4 splat textures result in one control texture; 5 splat textures result in two control textures.

  8. Set the expected control texture size, all control textures will use the same size. This size determines the quality of the painted texture: bigger size means better quality but more memory will be consumed.
  9. Then set the expected resolution of the mesh-terrain. This will determine the vertex count of the mesh-terrain. For example, if the resolution is 32, there will be 32*32 vertexes in the generated mesh-terrain. It’s recommended to use the same resolution with your original mesh.
  10. Press Begin Convert at the bottom and wait a moment.
  11. When it is done, files of the mesh-terrain(including a mesh obj file, a mat file, several control texture file, a prefab file - if you have chosen Create prefab) is placed in the output dir. And the instanced prefab(if you have chosen Create prefab) or the mesh object is in the hierarchy.
  12. Set splat textures into the material(s) of the mesh-terrain object.

Then you can use the MTE Editor to edit the generated mesh-terrain.

Editing an existing mesh (experimental feature)

You can edit the heights of an existing mesh directly as long as the mesh is in obj format. If it's not, you need to convert it to obj.

If you want to paint textures on an existing mesh, please follow these steps:

  1. Choose a shader provided by MTE: MTE/N Textures/XXX. You CAN use your own shader, but there are some conventions to follow. See FAQ: Can I change the shader? for more detail.
  2. Manually create one or two control texture(s) and assign it to the material. For a mesh with 2/3/4 splat textures, only 1 control texture is needed. But for a mesh with 5 splat textures, 2 control textures are needed.
    You can do that by using a tool such as PhotoShop. The control textures need to follow several conventions:
    1. They are the same size.
    2. The width and height should be the same and power of 2 (32/64/128/256/512/1024).
    3. The format is RGBA 32bit depth.
    4. The first control texture is all red: the R channel is 1.0 and all others are 0.0. The possible second control texture is all black: all channels are 0.0.
    5. In the importer settings of the control textures, choose Advanced -> Read/Write Enabled .

Editing Mesh-terrains

The height, texture and vertex color is editable for a mesh-terrain. MTE also provided two simple tools to move vertex sideways and cut holes on the mesh.

Height Tools

You can use first three height tools to modify height of the mesh-terrain.

  • Raise and lower

    Use mouse to raise and lower height in the scene view. Press Space to toggle between raising up and lowering down.

    You can modify the size and strength of the brush in the editor window.

    You can also use -/+ to decrease/increase the size of the brush and [/] to decrease/increase the strength.

    You can adjust the falloff curve using the curve editor.

  • Paint height

    Hold Shift and move mouse to sample a height. And use mouse to paint height in the scene view.

    The height can also be specified directly in the editor window.

    You can modify the size and speed of the brush in the editor window.

    You can also use -/+ to decrease/increase the size of the brush and [/] to decrease/increase the speed.

  • Smooth height

    Use mouse to smooth height in the scene view.

    You can modify the size and speed of the brush in the editor window.

    You can also use -/+ to decrease/increase the size of the brush and [/] to decrease/increase the speed.

Warning! You must save the mesh terrain by clicking the Save button below! Ctrl+S WON’T work!

Paint textures

Select a brush and a texture, then click to draw in the scene view.

You can modify the size and flow of the brush in the editor window; also you can use -/+ to increase the size of the brush and [/] to decrease/increase the flow.

MTE also provided two tools here:

  • bake basemap
    Bake the material into a png picture. The basemap png file will be generated next to the mat file of the mesh-terrain.
  • convert to Unity3D terrain
    Convert the mesh-terrain to Unity3D terrain and retain the splat and control(alpha) textures. The Unity3D terrain asset file will be output to the root project directory.

Modification will be saved instantly.

Paint vertex colors

At the beginning, you need to add vertex color component to the gameObject with mesh.

If the vertex color component is ready. You can select a color, then click to draw in the scene view.

Choose the eraser to erase the vertex color to white(R:1.0 G:1.0 B:1.0 A:1.0)

You can modify the size and speed of the brush in the editor window; also you can use -/+ to increase the size of the brush and [/] to decrease/increase the flow.

You can adjust the intensity to get a lighter color than the current color.

Warning! Do not check Optimize Mesh in the mesh import settings, even if you have finished editing the vertex colors. The optimization by the mesh importer will possibly change the vertex count and makes the vertex colors not the same number as the vertex. This causes problems when editing and saving the vertex colors in the vertex color asset file.

Vertex colors will be saved instantly.

Mesh Toolbox

Mesh toobox is not provided in free version.

First click on the red light of the tool below to enable it.

  • Move vertex sideways
    1. Click on a triangle to select it.
    2. Press Tab to choose a vertex.
    3. Use the position handle to move the vertex.
    4. Press Esc to unselect a triangle.
  • Cut holes

    Click on a triangle to remove it.

    Note Isolated vertexes in the mesh will not be removed.

    Note You cannot cut holes on a vertex-colored mesh terrain.

    Note After saving the mesh, MTE is unable to fill the hole.

Warning! You must save the mesh terrain by clicking the Save button below! Ctrl+S WON’T work!

Undo and Redo

All editing operations on the mesh or textures are undoable. Use PageUp to undo and PageDown to redo. Changes to the parameters on GUI are not undoable.

Note Undo/redo data will be cleared after you saving the mesh.

Settings

The final tool on the toolbar is settings:

  • Show affected vertexes: check this if you want to display the dots representing the vertexes being modifying.
    • Flash affected vertexes: check this if you want to make the dots flash
    • Affected vertexes size: adjust the size of the dots
  • Show hit point: check this if you want to see the point where the mouse is hitting.

Customize UI Language

You can adjust the language and text content of the GUI. Just Add a StringTable.cs to your project (in a “Editor” directory or sub-dir of a “Editor” dir). StringTables of English and Chinese are already provided: Editor Default Resources\StringTable.en.txt, Editor Default Resources\StringTable.cn.txt.

The default language is English.

If you want to customize the GUI, follow those steps:

  1. Copy StringTable.en.txt and rename it to StringTable.cs. (Any name is OK as long as it is a cs file.)
  2. Modify the text inside the second pair of double quote. For example, modify {"Warning.Confirm", "Are you sure?"}, to {"Warning.Confirm", "You need to confirm this operation! Are you sure?"},

    Just modify those you need. Any text that are not specified will fallback to English.

  3. Put the StringTable.cs next to the MTE.dll

FAQ

  1. I didn't see the Window/MTE in the menu！

    Please open the importer setting of Plugins/Editor/MTE/MTE.dll. Then check if the Platform Settings->CPU is AnyCPU or is correpond to the using Unity3D. If you're not sure, just try changing that.

  2. Why pressing Ctrl+Z will not redo?

    Use PageUp to undo and PageDown to redo. These keys are used instead of the famous Ctrl+Z because the undo system provided by Unity3D is just TOO SLOW for big data such as a big array. I’ll try to make Ctrl+Z/Y work in future versions, maybe.

  3. I cannot edit using the height tool. Not seeing any flashing vertex.

    Try disabling and enabling the MeshCollider component. We still cannot figure out why it happens and why this works. I only find that happens when you reimport/save the mesh-terrain obj file.

  4. Can I change the shader?

    Sure. And all editing functions have been decupled from the shaders provided by MTE. But if you want to continue editing the mesh-terrain in MTE, you must follow conventions below.

    • To use the height tools(raise/lower, paint height, smooth height), make sure that the object has a valid mesh and MeshCollider attached.
    • To use the paint textures tool, make sure that the shader contains a Texture2D property named _Control and it is all red in the beginning. If you use a second control texture, the property name of it must be _ControlExtra and it is all black in the beginning. But I suggest you use the shaders provided by MTE as templates. The splat textures' property name should be _Splat0/1/2/3/4. If you are using a bumped shader, the property name of the normal textures should be _Normal0/1/2/3/4. We suggest you use the shaders provided by MTE and modify them as you see fit.
    • To use the paint vertex color tool, make sure that the gameObject selected have a VertexColorInitializer component attached. (This script component is automatically attached to the gameObject using the button on the editor window. See the description of this tool.)
    • To use the mesh toolbox/Cut holes tool, make sure that the gameObject selected does NOT have a VertexColorInitializer component attached: you cannot cut holes on a vertex-colored mesh-terrain.

  5. Hotkey not working!

    Does the scene view or the MTE editor has the focus? If it is not, the hotkey will not work.

    Do you close the IME? It’s the IME that is disturbing the hotkey handling routine and makes it not work. Close it.

  6. I cannot unselect the mesh-terrain. / I cannot select other gameObjects.

    Please select other gameObject in the hierarchy window. MTE locked the selection in the scene view window to forbid misoperation when editing the mesh-terrain.

  7. What is done when adding vertex color component?

    This operation will create a asset file of the vertex colors aside the mesh file. And a VertexColorInitializer component will be attached to the gameObject.

    The component is used to apply vertex colors in the asset file to the Mesh.colors when the gameObject is created. The source codes of the component and the asset are provided here: Assets\Editor Default Resources\MTE\MTE.zip. Those two cs file is already compiled into a dll here: Assets\Plugins\MTE\MTE_script.dll. The source codes are provided so you can modify them and compile them as MTE_script.dll and then replace the original one if you want to. Don't extract them into your project directory.

  8. Why MTE Editor is closed when in play mode?

    Editing in play mode is not tested yet. So MTE editor is just closed before entering play mode to restrict possible misoperations.

  9. Why there is a Material folder in the output dir when conversion finished?

    It’s generated by Unity for previewing the mesh file in the inspector. You can safely remove it yourself.

Known Issues

  1. CPU occupancy rate is high when using height tools for some big mesh-terrain.

    The hit point of mouse is generated using Physics.Raycast, so the MeshCollider needs to be updated each time the Mesh is modified. It is the process of updating a collider, which is handled by Unity3D internally, that consumes much CPU time when vertex count of a mesh is big. This problem sometimes happened when editing a regular mesh but we still cannot find the cause. in some If you run into this problem, please try reopen your project.

用户手册

模型地形编辑器（Mesh Terrain Editor，MTE）有两个主要功能：

  • MTE转换器：用于将MTE的地形对象、一般模型转换为MTE可编辑的模型。（现在MTE也可编辑一般模型了——实验性功能）（部分功能在免费版有限制）
  • MTE编辑器：用于编辑MTE可编辑的模型，包含高度工具、纹理绘制工具、顶点颜色绘制工具、模型修改工具（自由移动顶点和挖洞）。（部分功能在免费版有限制或不可用）

后文中MTE可编辑模型简称为mesh-terrain.

创建mesh-terrain

您可以将模型或Unity3D地形转换为mesh-terrain，mesh-terrain只是一个使用了特殊地形shader的模型。 MTE提供了很多相关的shader。

将Unity地形转换为mesh-terrain

如果您想将现有的Unity地形转换为模型，并使用MTE编辑，请按照以下步骤操作：

  1. 通过菜单 Window/MTE/Converter, 打开MTE转换器，然后点击 地形 按钮.
  2. 选中需要转换的地形对象。注意，地形的贴图数量需要是2/3/4/5，才能进行转换。在免费版中您只能使用4张贴图。
  3. 输入转换后的模型的名字。
  4. 设置输出文件夹路径。此路径必须在工程文件夹下的Assets目录中。
  5. 点击底部的 开始转换 按钮，稍等一会儿。
  6. 转换完成后，mesh-terrain的相关文件会生成到输出目录中的一个命名为模型名的文件夹中。其中的模型对象的预置体会被实例化到场景中。

然后您就可以使用MTE编辑器编辑生成的mesh-terrain了。

注意 MTE已经为您调整贴图的 Tilling 以符合原地形对象的样子，但是您可以根据需要在任何时候调整各贴图的 Tilling 值。

将一般模型转换为mesh-terrain

如果您想将已有模型（如从3ds Max中导出的fbx模型）转换为mesh-terrain，并使用MTE进行编辑，请按照以下步骤操作：

  1. 导入您的模型文件到工程中，将模型拖动到场景中。
  2. 通过菜单 Window/MTE/Converter, 打开MTE转换器，然后点击 模型 按钮。
  3. 选中要转换的地形模型，注意要选中模型本身（带有MeshRenderer的那个）而不是其父节点
  4. 输入转换后的模型的名字。
  5. 设置输出文件夹路径。此路径必须在工程文件夹下的Assets目录中。
  6. 如果需要将转换后的mesh-terrain保存为预置体，请选中 创建预置体 。否则转换后的mesh-terrain将直接保存在场景中。推荐选中。
  7. 设置需要的贴图数量。只能使用 2/3/4/5 张贴图。免费版只能选择使用4张贴图。

    注意：贴图数量在转换后可以变化，可通过切换shader做到。具体情况比较多，请自行尝试。

    注意：控制图的实际数量由贴图数量决定。对于2/3/4张贴图，转换后控制图的数量为1；对于5张贴图，转换后的控制图的数量为2.

  8. 设置期望的控制图大小，所有的控制图都是统一大小。 这个大小会影响绘制贴图的质量，更大的尺寸意味着更好的质量，但是也会占用更多的空间。
  9. 设置mesh-terrain的分辨率。这会影响mesh-terrain的顶点数量：如果分辨率是32，生成的模型就会有 32*32=1024 个顶点。推荐使用和原模型类似的分辨率。
  10. 点击底部的 开始转换 按钮，稍等一会儿。
  11. 转换完成后，mesh-terrain的相关文件（包括一个模型obj文件、mat材质文件、一个或两个控制图文件和一个预置体文件——如果之前选中了创建预置选项）会生成到输出目录中的一个命名为模型名的文件夹中。如果之前选中了创建预置选项，预置体会被实例化到场景中。
  12. 设置需要的贴图到mesh-terrain的材质中

然后您就可以使用MTE编辑器编辑生成的mesh-terrain了。

直接编辑一般模型（实验性功能）

您可以直接编辑一般模型的高度，只需要将其模型文件转换为obj格式即可。

如果您需要编辑贴图，请按照以下步骤操作：

  1. 在模型的材质中，选用MTE提供的shader：MTE/N Textures/XXX。你可以使用自己的shader，但是有一些规则要遵守，详见 常问问题（可以替换shader吗？）。
  2. 手动生成1张或2张控制图。2/3/4张贴图需要1张控制图，5张贴图需要2张控制图。您可以使用图像编辑工具如PhotoShop来生成控制图。生成的控制图需要满足以下条件：
    1. 相同尺寸
    2. 长宽相等均为2的幂（32/64/128/256/512/1024）
    3. 格式为RGBA 32位深度
    4. 第一张控制图必须是纯红色（R通道为1.0），第二张控制图必须是全黑色（RGBA通道都是0.0）
    5. 在Unity3D的导入选项中，需要选中Advanced -> Read/Write Enabled

编辑mesh-terrain

mesh-terrain的高度、贴图、顶点颜色都可以利用MTE进行编辑。MTE也提供了两个简单的模型编辑功能：自由移动顶点、挖洞。

Height Tools

你可以点击前三个工具按钮来使用高度工具，高度工具用于修改mesh-terrain的顶点的y坐标（即高度）

  • 提升/降低高度

    拖动鼠标绘制，按 空格 键来切换上升和下降。

    您可以在编辑器窗口中修改笔刷大小 和 强度 。

    您也可以使用快捷键 - 或 + 来减少或增加笔刷 大小， [或] 来减少或增加笔刷 强度。

    使用编辑器中的衰减曲线编辑器可以调整绘制时高度随距离笔刷中心的距离进行衰减的曲线

  • 绘制高度

    按住 Shift 键移动鼠标对 高度 取样。 拖动鼠标绘制。

    高度 也可以在编辑器窗口中直接设置。

    您可以在编辑器窗口中修改笔刷大小 和 速度 。

    您也可以使用快捷键 - 或 + 来减少或增加笔刷 大小， [或] 来减少或增加笔刷 速度。

  • 平滑高度

    拖动鼠标来平滑高度。

    您可以在编辑器窗口中修改笔刷大小 和 速度 。

    您也可以使用快捷键 - 或 + 来减少或增加笔刷 大小， [或] 来减少或增加笔刷 速度。

警告！编辑完成后必须点击下方的“保存”按钮才会保存，使用Ctrl+S没有任何效果

绘制贴图

选中一个笔刷和要绘制的贴图，然后点击拖动鼠标来绘制贴图。

您可以在编辑器窗口中修改笔刷大小 和 流量 。

MTE也在此提供了两个工具：

  • 生成Basemap
    将mesh-terrain的材质转换为一张PNG图像。生成的图像文件会放置在mat材质文件旁边。
  • 转换为Unity3D地形
    将mesh-terrain转换为Unity3D地形，保留贴图和控制图。生成的Unity3D地形对象的asset文件将被输出到Assets目录下。

对控制图的修改会马上被保存，无需手动进行保存。

绘制顶点颜色

刚开始时，您需要添加顶点颜色相关组件到模型的GameObject上，点击出现的按钮即可。

如果添加过了，请再确保你的shader反应了顶点颜色的变化（MTE提供了顶点颜色相关的shader：MTE/N Textures/XXX 或 MTE/VertexColored/XXX ）。此mesh-terrain的顶点颜色就可被编辑了。选择一个颜色，拖动鼠标即可绘制。

选择橡皮擦来擦除颜色为白色（R:1.0 G:1.0 B:1.0 A:1.0）

您可以添加颜色到颜色预置当中，点击箭头即可；点击预置中的颜色方块可设置当前颜色为预置的颜色。

您可以选择使用单通道绘制，选择后，再选择通道RGBA，之后的绘制就只会对顶点颜色的此通道产生影响。

您可以在编辑器窗口中修改笔刷大小 和 流量 。

你可以调整笔刷 硬度 来使用一个比当前颜色更亮的颜色。1.0表示当前颜色。

警告！在模型的导入选项中，不要选中 Optimize Mesh。 否则Importer对模型的很可能会更改顶点数量和索引顺序。此类优化会导致模型数据和文件中数据的不一致，进而导致在使用MTE编辑顶点颜色时，顶点颜色的数量和顶点数量不一致。这会在保存或加载顶点颜色时，引起顶点颜色的错乱。并且，因为类似的原因，已经使用了顶点颜色的mesh-terrain无法被挖洞。

对顶点颜色的修改会马上被保存，无需手动进行保存。

模型工具

模型工具在免费版中不可用.

首先点击红色灯来激活需要的工具。

  • 自由移动顶点
    1. 点击选取三角形
    2. 按Tab来选择顶点。
    3. 使用出现的handle移动顶点。
    4. 按Esc取消三角形选择
  • 挖洞

    点击三角形来删除它

    注意 孤立顶点没有从obj文件中删除。

    注意 在使用了顶点颜色的mesh-terrain不能挖洞。

    注意 保存模型后，MTE不能撤销之前的挖洞操作，也不能填上挖掉的洞。

警告！编辑完成后必须点击下方的“保存”按钮才会保存，使用Ctrl+S没有任何效果。

撤销和重做

所有在模型和贴图上的编辑操作，都是可以撤销重做的。使用 PageUp 进行撤销， PageDown 进行重做。编辑器上参数的修改无法撤销。

注意 撤销重做的数据会在你保存模型时被清空。

设置

工具栏的最后一个工具是设置：

  • 标识受影响的顶点： 选中此项后会显示正在被编辑的顶点。
    • 闪烁：选中此项后会闪烁标识点。
    • 标识大小: 修改标识点的大小
  • 显示碰撞点: 选中此项后会显示当前鼠标的碰撞点

自定义UI语言

您可以自行调整界面的语言和文字内容。为此，需要添加一个StringTable.cs文件，此文件的格式请参考MTE提供的英文和简体中文的文件：Editor Default Resources\StringTable.en.txt, Editor Default Resources\StringTable.cn.txt.

默认语言是英文。

如果需要修改界面显示的文本内容，请按照以下步骤进行：

  1. 复制 StringTable.xx.txt 并重命名为 StringTable.cs.
  2. 修改其中的文本内容，位于第二个引号组内的。例如，
    {"Warning.Confirm", "确定执行此操作吗?"},
    可以被修改为
    {"Warning.Confirm", "你需要确认执行此操作！确定吗？"},

  3. 将 StringTable.cs 放在 MTE.dll 的旁边

常问问题

  1. 我找不到Window/MTE菜单！

    请确认 Plugins/Editor/MTE/MTE.dll 的导入选项 Platform Settings->CPU 中，是否是Any CPU或者和您的Unity3D版本相同。x86版本的Unity3D只能使用x86导入选项的MTE.dll，x64版本的Unity3D只能使用x64导入选项的MTE.dll. 如果你不确定，修改几次试试。

  2. 为何按 Ctrl+Z 不会撤销？

    使用 PageUp 来撤销，使用 PageDown 来重做。 因为Unity3D的撤销机制对于地形模型这种大对象而言，太慢了！所以MTE采用了自行编写的一套撤销系统，用于提升用户体验，为了不和Unity3D的快捷键发生冲突，就采用。未来也许会尝试让 Ctrl+Z/Y 起作用，但是可能性不大。

  3. 我无法使用高度工具。也没有看到任何闪烁的顶点。

    请尝试关闭MeshCollider组件后再打开它。目前还不清楚为什么会发生这种情况，以及为什么这样做有效，只是发现无法编辑的情况通常在保存或reimport模型文件后发生。

  4. 我可以更改shader吗？

    当然可以。 所有的编辑功能已经和MTE提供的shader解绑了。但是为了继续在MTE中编辑mesh-terrain，您需要遵循以下的规则：

    • 为了可以使用 高度工具（提升/降低高度、绘制高度、平滑高度）以及模型工具，请确保对象上有有效的MeshCollider组件。
    • 为了可以使用 绘制贴图 工具，确保使用的shader中的控制图的属性名为_Control；如果使用了第二张控制图，其对应的属性名应为_ControlExtra. 第1/2/3/4/5层贴图的属性名应为 _Splat0/1/2/3/4. 如果使用的是bumped shader，法线贴图的属性名应为_Normal0/1/2/3/4. 建议参考MTE提供的shader进行修改。
    • 为了可以使用 顶点颜色绘制 工具，确保对象上有VertexColorInitializer组件（此组件会在编辑器中确认添加顶点颜色时，自动添加，更多相关内容参见 常问问题：添加顶点颜色组件时，发生了什么？）
    • 为了可以使用 模型工具/挖洞 tool, 请确保对象上没有颜色绘制组件，您不能在绘制了顶点颜色的模型上挖洞。

  5. 快捷键无效！

    请检查场景窗口（Scene）或MTE编辑器窗口有焦点吗？如果没有，快捷键不会有作用。

    输入法关闭（切换到英文）了吗？输入法会扰乱快捷键的处理从而让它不起作用。请关掉输入法。

  6. 我不能取消选择mesh-terrain. 或我不能选中其他对象了。

    请在hierarchy窗口中选择其他对象。编辑时，MTE在场景窗口中锁定了选择的对象来防止误操作。

  7. 添加顶点颜色组件时，发生了什么？

    此操作会在模型obj文件旁生成一个asset文件用于保存顶点颜色。一个VertexColorInitializer组件会被附加到模型对象上。

    此组件用于在模型的GameObject创建时将asset文件中的顶点颜色赋值给Mesh.colors. 相关类的源码在这里：Assets\Editor Default Resources\MTE\MTE.zip 相关类的已经被编译为dll了，在这里：Assets\Plugins\MTE\MTE_script.dll. 提供源码是为了让您可以根据需要修改他们并编译后替换原来的MTE_script.dll，所以不要直接将其解压到您的工程目录中

  8. 为什么MTE编辑器在运行时被关闭了？

    在运行时编辑还没有被测试过。所以MTE编辑器在开始运行时关闭了以防止可能的误操作。

  9. 为什么在转换完成后，输出目录中有一个Material文件夹？

    此文件夹是由Unity3D生成的，用于在inspector中预览模型文件。您可以自行删掉它。

已知问题

  1. 在编辑较大mesh-terrain时CPU占用率偏高

    原因是，编辑过程中鼠标的碰撞点是通过Physics.Raycast生成的，所以MeshCollider需要在模型顶点改变时进行更新，更新的过程完全由Unity3D内部的物理引擎控制，对于一些大的模型，此过程非常耗时。如果是拖动编辑则会出现严重的卡顿现象。此问题在常规顶点数的模型上也有发生，还没有确定原因，如果您遇到此问题，可以尝试重启Unity3D，一般可以解决。

2016-6-10 v2.2

  • 增加两种类型的shader。SinglePass模式：占用1个DrawCall，可用2~5张贴图。推荐用于移动平台；Layered模式：占用1~3个DrawCall，可用1~12张贴图。
  • 相关转换、编辑功能更新

2016-5-28 v2.1

  • 添加临时功能：将5张贴图的地形对象转换为5张贴图的模型，模型渲染只占用一个DrawCall，并且使用target 2.0. 转换后的Mesh同样可直接使用MTE 2.1进行编辑。

2016-5-14 v2.0

  • 采用四叉树算法优化了编辑时性能，减少卡顿
  • 添加“模型工具箱”，包含两个功能：xyz方向自由移动顶点、挖洞
  • 增强对贴图数量的支持，现在支持多达12张贴图（3张控制图）的绘制
  • Shader改进，采用了Surface shader作为默认shader（原为纯CG），渲染效果改进
  • Personal版 Unity3D中的界面改进
  • 修复快捷键bug：编辑器窗口有焦点时，快捷键也无效
